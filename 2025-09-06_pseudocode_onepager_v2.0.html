Grundlagen & Syntax

Pseudocode vs. C# - Die wichtigsten Unterschiede

Notation und Schreibweise

Pseudocode ist wie C# ohne Klammern: Während C# geschweifte Klammern {...} für Codeblöcke verwendet, nutzt Pseudocode deutsche Begriffe wie "Ende wenn" oder "Ende solange".

Wichtig für die Prüfung: Saubere Einrückungen sind entscheidend für die Punktevergabe! Die Prüfer erwarten korrekt eingerückte Blöcke, genau wie in C#.

C#:
if (alter >= 18) {
    Console.WriteLine("Erwachsen");
}
Pseudocode:
wenn alter >= 18 dann
    Ausgabe "Erwachsen"
Ende wenn

Kommentare

Identisch zu C#:

Einzeilig: // Das ist ein Kommentar
Mehrzeilig: /* Das ist ein
    mehrzeiliger Kommentar */

Strukturregeln

  • Blöcke: Statt {...} verwendet Pseudocode deutsche Wörter (Ende wenn, Ende solange)
  • Einrückung: Wie in C# - macht den Code lesbar
  • Groß-/Kleinschreibung: Weniger streng als C#, aber konsistent bleiben

Datentypen

Primitive Datentypen - Die gleichen Konzepte, andere Namen

Boolean (Wahrheitswerte)

Identisch zu C#: Speichert entweder True oder False. Im IHK-Pseudocode werden die englischen Begriffe verwendet, genau wie in C#.

C#:
bool istErwachsen = true;
Pseudocode:
istErwachsen: Boolean
istErwachsen := True

Integer (Ganzzahlen)

Wie C# int: Speichert ganze Zahlen wie 42, -17, 0. Der Pseudocode verwendet den ausgeschriebenen Begriff "Integer" statt der C# Kurzform "int".

C#:
int alter = 25;
Pseudocode:
alter: Integer
alter := 25

Character (Einzelne Zeichen)

Wie C# char: Speichert ein einzelnes Zeichen wie 'A', '1', '+'. Wird in einfachen Anführungszeichen geschrieben.

C#:
char note = 'A';
Pseudocode:
note: Character
note := 'A'

String (Zeichenketten)

Identisch zu C#: Speichert Text wie "Hallo Welt". Wird in doppelten Anführungszeichen geschrieben, genau wie in C#.

C#:
string nachname = "Müller";
Pseudocode:
nachname: String
nachname := "Müller"

Double (Gleitpunktzahlen)

Wie C# double: Speichert Dezimalzahlen wie 3.14, -2.5, 0.0. Wichtig für Berechnungen mit Nachkommastellen.

C#:
double preis = 19.99;
Pseudocode:
preis: Double
preis := 19.99

Void (Kein Rückgabetyp)

Identisch zu C# void: Wird bei Methoden verwendet, die keinen Wert zurückgeben. Das Konzept ist exakt das gleiche wie in C#.

C#:
public void SageHallo() {
    // Code hier
}
Pseudocode:
+ SageHallo() : Void
    // Code hier
Ende SageHallo

Variablen & Zuweisungen

Variablen & Zuweisungen - Der wichtigste Unterschied zu C#

Getrennte Deklaration und Zuweisung

Der Hauptunterschied: Während C# Deklaration und Zuweisung in einer Zeile erlaubt, muss im Pseudocode erst deklariert und dann separat zugewiesen werden.

Wichtig für die Prüfung: Diese Trennung ist Pflicht! Kombinierte Deklaration wie in C# führt zu Punktabzug.

C# (kombiniert):
int alter = 25;
string name = "Max";
Pseudocode (getrennt):
alter: Integer
alter := 25
name: String
name := "Max"

Zuweisungsoperator := vs =

Neues Symbol lernen: Pseudocode verwendet := für Zuweisungen, während C# das einfache = nutzt. Das = ist in Pseudocode nur für Vergleiche reserviert.

C#:
punkte = 100;
punkte = punkte + 50;
Pseudocode:
punkte := 100
punkte := punkte + 50

Variablendeklaration Syntax

Reihenfolge umgekehrt: C# schreibt den Typ zuerst, Pseudocode den Variablennamen zuerst. Das ist anfangs verwirrend, aber mit etwas Übung wird es zur Gewohnheit.

C# Format:
double temperatur;
bool istAktiv;
char zeichen;
Pseudocode Format:
temperatur: Double
istAktiv: Boolean
zeichen: Character

Lokale Variablen zur Laufzeit

Flexibilität wie in C#: Variablen können auch mitten im Code deklariert werden, nicht nur am Anfang. Das ist besonders nützlich für temporäre Hilfsvariablen.

C#:
if (a > b) {
    int temp = a;
    a = b;
    b = temp;
}
Pseudocode:
wenn a > b dann
    temp: Integer
    temp := a
    a := b
    b := temp
Ende wenn

Operatoren

Operatoren - Meist identisch, ein paar Ausnahmen beachten

Vergleichsoperatoren

Fast alle gleich: Die meisten Vergleichsoperatoren funktionieren genauso wie in C#. Eine Ausnahme: <> für "ungleich" statt C#s !=.

C#:
alter == 18 // gleich
alter != 18 // ungleich
alter > 18 // größer
alter >= 18 // größer gleich
Pseudocode:
alter = 18 // gleich
alter <> 18 // ungleich
alter > 18 // größer
alter >= 18 // größer gleich

Logische Operatoren

Andere Symbole: Pseudocode verwendet Buchstaben statt Symbole für logische Operationen. Das macht den Code lesbarer, ist aber anfangs ungewohnt.

C#:
alter >= 18 && fuehrerschein
wochenende || urlaub
!regen
Pseudocode:
alter >= 18 && fuehrerschein
wochenende OR urlaub
NOT regen

Mathematische Operatoren

Grundrechenarten identisch: Addition, Subtraktion, Multiplikation funktionieren gleich. Bei der Division gibt es einen wichtigen Unterschied zwischen Ganzzahl- und Komma-Division.

C#:
summe = a + b;
differenz = a - b;
produkt = a * b;
quotient = a / b;
rest = a % b;
Pseudocode:
summe := a + b
differenz := a - b
produkt := a * b
quotient := a / b
rest := a % b

Besondere Operatoren

Zeichenketten-Verkettung: Pseudocode verwendet & zum Verbinden von Strings, C# nutzt +. Das & macht deutlich, dass es sich um Textverarbeitung handelt.

C#:
vollName = vorname + " " + nachname;
meldung = "Hallo " + name + "!";
Pseudocode:
vollName := vorname & " " & nachname
meldung := "Hallo " & name & "!"

Operator-Prioritäten

Gleiche Regeln wie C#: Punkt vor Strich, Klammern zuerst, logische Operatoren haben niedrigere Priorität als Vergleiche. Die mathematischen Regeln gelten unverändert.

C#:
ergebnis = a + b * c;
ergebnis = (a + b) * c;
Pseudocode:
ergebnis := a + b * c
ergebnis := (a + b) * c

Ein-/Ausgabe

Ein-/Ausgabe - Einfacher als C# Console-Befehle

Ausgabe (Text ausgeben)

Viel einfacher als C#: Statt Console.WriteLine schreibt man einfach Ausgabe. Das spart Tipparbeit und macht den Code lesbarer.

Wichtig für die Prüfung: Anführungszeichen nicht vergessen! Text steht immer in Anführungszeichen, Variablen ohne.

C#:
Console.WriteLine("Hallo Welt!");
Console.WriteLine(name);
Console.WriteLine("Alter: " + alter);
Pseudocode:
Ausgabe "Hallo Welt!"
Ausgabe name
Ausgabe "Alter: " & alter

Eingabe (Benutzereingabe lesen)

Direkter als C#: Eingabe liest automatisch einen Wert ein und weist ihn der Variable zu. Kein Console.ReadLine() und keine Typkonvertierung nötig.

Automatische Typerkennung: Pseudocode erkennt selbst, ob eine Zahl oder Text eingegeben wird - anders als in C#, wo alles erstmal String ist.

C#:
string eingabe = Console.ReadLine();
int zahl = int.Parse(eingabe);
Pseudocode:
Eingabe name
Eingabe alter

Kombinierte Ein-/Ausgabe

Benutzerfreundliche Eingabe: Erst eine Aufforderung ausgeben, dann die Eingabe lesen. Das macht Programme verständlicher für den Benutzer.

C#:
Console.Write("Geben Sie Ihr Alter ein: ");
int alter = int.Parse(Console.ReadLine());
Console.WriteLine("Sie sind " + alter + " Jahre alt.");
Pseudocode:
Ausgabe "Geben Sie Ihr Alter ein: "
Eingabe alter
Ausgabe "Sie sind " & alter & " Jahre alt."

Mehrere Werte ausgeben

Flexible Ausgabeformate: Variablen und Text können beliebig mit dem & Operator kombiniert werden. Das entspricht der String-Verkettung in C#, ist aber übersichtlicher.

C#:
string vorname = "Max";
string nachname = "Mustermann";
int punkte = 95;
Console.WriteLine(vorname + " " + nachname + " hat " + punkte + " Punkte erreicht.");
Pseudocode:
vorname: String
vorname := "Max"
nachname: String
nachname := "Mustermann"
punkte: Integer
punkte := 95
Ausgabe vorname & " " & nachname & " hat " & punkte & " Punkte erreicht."

Kontrollstrukturen

Kontrollstrukturen - Deutsche Begriffe statt Klammern

IF-THEN-ELSE (Verzweigungen)

Deutsche Schlüsselwörter: Statt if, else verwendet Pseudocode wenn, sonst. Das dann ist Pflicht und darf nicht vergessen werden!

Wichtig: Jede Verzweigung muss mit Ende wenn geschlossen werden. Einrückungen sind für die Punktevergabe entscheidend.

C#:
if (alter >= 18) {
    Console.WriteLine("Volljährig");
} else {
    Console.WriteLine("Minderjährig");
}
Pseudocode:
wenn alter >= 18 dann
    Ausgabe "Volljährig"
sonst
    Ausgabe "Minderjährig"
Ende wenn

CASE/SWITCH (Mehrfachauswahl)

Ähnliche Struktur: wenn entspricht switch, sonst wenn entspricht case. Der sonst-Block entspricht default.

Kein break nötig: Anders als in C# gibt es kein "Durchfallen" zwischen den Fällen - jeder Block wird automatisch beendet.

C#:
switch (note) {
    case 1:
        Console.WriteLine("Sehr gut");
        break;
    case 2:
        Console.WriteLine("Gut");
        break;
    default:
        Console.WriteLine("Andere Note");
        break;
}
Pseudocode:
wenn note = 1 dann
    Ausgabe "Sehr gut"
sonst wenn note = 2 dann
    Ausgabe "Gut"
sonst
    Ausgabe "Andere Note"
Ende wenn

WHILE-Schleife (solange)

Kopfgesteuerte Schleife: solange entspricht while. Die Bedingung wird vor jedem Durchlauf geprüft. Möglicherweise wird die Schleife gar nicht ausgeführt.

C#:
int i = 0;
while (i < 10) {
    Console.WriteLine(i);
    i++;
}
Pseudocode:
i: Integer
i := 0
solange i < 10
    Ausgabe i
    i := i + 1
Ende solange

FOR-Schleife (zähle)

Zählschleife: zähle entspricht der for-Schleife in C#. Start- und Endwert werden direkt angegeben, die Zählvariable wird automatisch erhöht.

Einfacher als C#: Keine komplizierte Syntax mit Initialisierung, Bedingung und Inkrement - alles in einer Zeile.

C#:
for (int i = 1; i <= 10; i++) {
    Console.WriteLine("Zahl: " + i);
}
Pseudocode:
zähle i von 1 bis 10
    Ausgabe "Zahl: " & i
Ende zähle

DO-WHILE (wiederhole)

Fußgesteuerte Schleife: wiederhole entspricht do-while. Die Schleife wird mindestens einmal ausgeführt, da die Bedingung erst am Ende geprüft wird.

C#:
int zahl;
do {
    Console.WriteLine("Zahl eingeben:");
    zahl = int.Parse(Console.ReadLine());
} while (zahl <= 0);
Pseudocode:
zahl: Integer
wiederhole
    Ausgabe "Zahl eingeben:"
    Eingabe zahl
solange zahl <= 0

Datenstrukturen

Datenstrukturen - Arrays als Sammlung gleichartiger Daten

Array-Deklaration

Andere Syntax: Pseudocode deklariert Arrays mit eckigen Klammern nach dem Typ, C# vor dem Variablennamen. Der Typ wird durch [] als Array gekennzeichnet.

Wichtig: Arrays speichern nur Elemente des gleichen Datentyps - genau wie in C#.

C#:
int[] zahlen;
string[] namen;
double[] preise;
Pseudocode:
zahlen: Integer[]
namen: String[]
preise: Double[]

Array-Initialisierung

Direkte Wertzuweisung: Arrays können direkt mit Werten gefüllt werden. Die geschweifte Klammern-Syntax {} ist in beiden Sprachen gleich.

C#:
int[] noten = {1, 2, 3, 2, 1};
string[] farben = {"rot", "grün", "blau"};
Pseudocode:
noten: Integer[]
noten := {1, 2, 3, 2, 1}
farben: String[]
farben := {"rot", "grün", "blau"}

Array-Zugriff und Indizierung

Gleiche Syntax: Der Zugriff auf Array-Elemente funktioniert identisch zu C#. Index beginnt bei 0, eckige Klammern für den Zugriff.

Wichtig für die Prüfung: Immer daran denken, dass Arrays bei Index 0 beginnen!

C#:
noten[0] = 1;
ersteFarbe = farben[0];
Console.WriteLine(noten[2]);
Pseudocode:
noten[0] := 1
ersteFarbe := farben[0]
Ausgabe noten[2]

Array-Länge (.length Eigenschaft)

Unterschiedliche Syntax: C# verwendet .Length (Großbuchstabe), Pseudocode .length (Kleinbuchstabe). Diese Eigenschaft gibt die Anzahl der Elemente zurück.

C#:
int anzahl = noten.Length;
for (int i = 0; i < noten.Length; i++) {
    Console.WriteLine(noten[i]);
}
Pseudocode:
anzahl: Integer
anzahl := noten.length
zähle i von 0 bis noten.length-1
    Ausgabe noten[i]
Ende zähle

Praktisches Array-Beispiel

Sortieralgorithmus: Hier sehen wir Arrays in der Praxis - ein einfacher Bubble Sort wie in deinen BFW-Unterlagen. Lokale Hilfsvariable wird mitten im Code deklariert.

C#:
for (int i = 0; i < zahlen.Length-1; i++) {
    if (zahlen[i] > zahlen[i+1]) {
        int temp = zahlen[i];
        zahlen[i] = zahlen[i+1];
        zahlen[i+1] = temp;
    }
}
Pseudocode:
zähle i von 0 bis zahlen.length-2
    wenn zahlen[i] > zahlen[i+1] dann
        temp: Integer
        temp := zahlen[i]
        zahlen[i] := zahlen[i+1]
        zahlen[i+1] := temp
    Ende wenn
Ende zähle

Funktionen & Methoden

Funktionen & Methoden - Modularer Code mit Parametern

Methodensignatur und Sichtbarkeit

Sichtbarkeitszeichen: Das + am Anfang entspricht public in C#. Die Methode ist von außen aufrufbar. Die Parameter stehen in Klammern, der Rückgabetyp nach dem Doppelpunkt.

Wichtig: Jede Methode muss mit Ende [Methodenname] geschlossen werden, nicht nur Ende.

C#:
public int AddierenZahlen(int a, int b) {
    return a + b;
}
Pseudocode:
+ AddierenZahlen(a: Integer, b: Integer) : Integer
    Rückgabe a + b
Ende AddierenZahlen

Void-Methoden (ohne Rückgabe)

Keine Rückgabe: Methoden mit Void geben keinen Wert zurück, sondern führen nur Aktionen aus. Ideal für Ausgaben oder Berechnungen ohne Ergebnis-Rückgabe.

C#:
public void BegruesseBEnutzer(string name) {
    Console.WriteLine("Hallo " + name + "!");
}
Pseudocode:
+ BegruesseBenutzer(name: String) : Void
    Ausgabe "Hallo " & name & "!"
Ende BegruesseBenutzer

Methodenaufruf

Gleiche Syntax: Der Aufruf einer Methode funktioniert identisch zu C#. Methodenname, dann Parameter in Klammern. Bei Void-Methoden kein Rückgabewert.

C#:
int summe = AddierenZahlen(5, 3);
BegruesseBenutzer("Anna");
Console.WriteLine(summe);
Pseudocode:
summe: Integer
summe := AddierenZahlen(5, 3)
BegruesseBenutzer("Anna")
Ausgabe summe

Lokale Variablen in Methoden

Gleiche Regeln: Variablen können in Methoden deklariert werden und sind nur dort gültig. Die getrennte Deklaration und Zuweisung gilt auch innerhalb von Methoden.

C#:
public double BerechneKreis(double radius) {
    double pi = 3.14159;
    double flaeche = pi * radius * radius;
    return flaeche;
}
Pseudocode:
+ BerechneKreis(radius: Double) : Double
    pi: Double
    pi := 3.14159
    flaeche: Double
    flaeche := pi * radius * radius
    Rückgabe flaeche
Ende BerechneKreis

Array-Parameter

Arrays als Parameter: Arrays können an Methoden übergeben werden. Die Syntax entspricht der Array-Deklaration mit [] nach dem Typ.

Praxisbeispiel: Diese Struktur kennst du aus dem Bubble Sort Algorithmus in deinen BFW-Unterlagen.

C#:
public int[] bubbleSort(int[] zahlen) {
    for (int i = 0; i < zahlen.Length-1; i++) {
        if (zahlen[i] > zahlen[i+1]) {
            // Tausch-Logik
        }
    }
    return zahlen;
}
Pseudocode:
+ bubbleSort(zahlen: Integer[]) : Integer[]
    zähle i von 0 bis zahlen.length-2
        wenn zahlen[i] > zahlen[i+1] dann
            // Tausch-Logik
        Ende wenn
    Ende zähle
    Rückgabe zahlen
Ende bubbleSort
